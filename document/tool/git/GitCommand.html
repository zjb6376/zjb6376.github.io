# git常用命令

  1. 查看git安装版本
git --version
  1. 初始化git仓库（会创建一个唯一的分支master）
git init
  1. 把文件添加到仓库（把文件添加到暂存区 .git）
git add 文件名    //git add hello.java 或者（添加该目录下所有的文件，用 . 表示全部文件）git add .
  1. 把文件提交到仓库（把暂存区的内容提交到当前分支）
git commit -m '备注'
  1. 查看仓库文件状态
git status
  1. 查看文件的变动
git diff
  1. 查看文件内容
cat 文件名
  1. 返回到上一个版本
git reset --hard HEAD^
  1. 从之前的版本再返回后面的版本
首先需要git log命令记录下后面版本的 commitid
然后 git reset --hard commitid
git reflog 可以查看历史命令，找回commitid
  1. 丢弃工作区的修改
修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。
git checkout -- 文件名    //git checkout -- hello.java
  1. 把暂存区的文件回退到工作区
git reset HEAD 文件名    //git reset HEAD hello.java
  1. 删除版本库的文件
git rm 文件名    //git rm demo.java
git commit -m "删除了demo文件"
  1. 从版本库恢复文件
git checkout -- 文件名
  1. 主目录下创建SSH Key
$ ssh-keygen -t rsa -C "956673039@qq.com"
一直回车完成
  1. 将本地库和远程github仓库关联起来
git remote add origin git@github.com:zjb6376/learngit.git
  1. 查看是否关联成功
ssh -T git@github.com
如果没有关联成功，修改在git的安装目录下/etc/ssh文件夹中的ssh_config文件
在最下面添加：
Host github.com
User 用户账号
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443
  1. 第一次推送到远程仓库
git push -u origin master
后续推送只需要 git push origin master
如果报错 ![rejected] master -> master (non-fast-forward)
执行 git pull origin master --allow-unrelated-histories
  1. 克隆远程仓库
git clone git@github.com:zjb6376/learn.git 或 git clone https://github.com/zjb6376/learn.git
  1. 创建分支branch
git branch 分支名称    //git branch dev
  1. 切换分支
git checkout 分支名    //git checkout dev        或者使用 git switch 分支名
  1. 查看所有分支
git branch
分支前带 * 代表当前所处的分支
  1. 合并分支到当前分支
git merge 分支名    //git merge dev
  1. 删除分支
git branch -d 分支名    //git branch -d dev
  1. 强制销毁分支
git branch -D 分支名
  1. 查看远程仓库详细信息
git remote -v
  1. git查看用户名和邮箱并修改
git config user.name
git config user.email
git config --global user.name "your name"
git config --global user.email "your email"
  1. 删除已连接的远程仓库
git remote rm origin
  1. 如果本地仓库要连到 github 和 gitee
git remote add github git@github.com:zjb6376/learn.git
git remote add gitee git@github.com:zjb6376/learn.git

git push github master
git push gitee master
  1. 忽略特殊文件
工作区根目录下创建 .gitignore 文件
*.class

多人协作的工作模式通常是这样：

  1. 首先，可以试图用git push origin 推送自己的修改；
  2. 如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；
  3. 如果合并有冲突，则解决冲突，并在本地提交；
  4. 没有冲突或者解决掉冲突后，再用git push origin 推送就能成功！

如果git pull提示no tracking information，则说明本地分支和远程分支的链接关系没有创建，用命令git branch --set-upstream-to origin/。

小结

查看远程库信息，使用git remote -v； 本地新建的分支如果不推送到远程，对其他人就是不可见的；从本地推送分支，使用git push origin branch-name，如果推送失败，先用git pull抓取远程的新提交； 在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；建立本地分支和远程分支的关联，使用git branch --set-upstream branch-name origin/branch-name；从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。
